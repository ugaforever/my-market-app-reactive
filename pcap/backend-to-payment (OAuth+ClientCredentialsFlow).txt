Описание основных этапов авторизации backend в payment-service (service-to-service) через Keycloak.

9090 backend
9091 payment-service
9092 keycloak


1. Пользователь оформляет заказ
POST "/buy"

2. Backend запрашивает токен у Keycloak (Client Credentials Flow)
POST http://localhost:9092/realms/my-market-app/protocol/openid-connect/token
	Backend (как клиент payment-service) запрашивает токен
	grant_type	client_credentials
	Client ID	payment-service
	Scope		openid

3. Keycloak выдает токен
Decoded [{access_token=eyJhbGciOiJSUzI1NiIsInR5cCIgOiAiSldUIiw...]
	Keycloak проверяет client_id и client_secret
	Возвращает access_token (JWT)
	Токен будет использован для авторизации запросов к payment-service

4. Backend запрашивает баланс пользователя
GET http://localhost:9091/api/v1/balance/4aafaab3-4a3d-4ab1-a4ad-3125fac6aecd
	Backend отправляет GET запрос к payment-service
	В заголовке автоматически добавлен Authorization: Bearer <token>
	AccountId = UUID пользователя из Keycloak (например 4aafaab3-4a3d-4ab1-a4ad-3125fac6aecd для testuser, такая моя реализация)

5. Payment-service возвращает баланс
Decoded [class BalanceResponse {
    accountId: 4aafaab3-4a3d-4ab1-a4ad-3125fac6aecd
    balance: 4667
    currency: RUB
}]
	Payment-service проверил токен
	Нашел счет пользователя в памяти (в учебных целях ConcurrentHashMap, в микросервисной архитектуре это будет сторонний сервис)
	Вернул баланс: 4667 RUB

6. Backend отправляет запрос на списание средств
POST http://localhost:9091/api/v1/payments
	Encoding [class PaymentRequest {
	    accountId: 4aafaab3-4a3d-4ab1-a4ad-3125fac6aecd
	    amount: 666
	}]
	Backend отправляет POST запрос на списание 666 RUB
	Токен автоматически добавляется в заголовок

7. Payment-service обрабатывает платеж
	Decoded [class PaymentResponse {
	    transactionId: 9b75353a-ab8e-4930-9c7c-e894fbb1de17
	    newBalance: 4001
	    status: SUCCESS
	}]
	Payment-service списал 666 RUB с баланса
	Новый баланс: 4001 RUB (4667 - 666 = 4001)
	Сгенерирован transactionId

8. Редирект на страницу заказа
GET "/orders/1?newOrder=true"
